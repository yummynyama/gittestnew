Универсальный каркас под **Модуль 1** (разработка ИС) и **Модуль 2** (дизайн) ГИА ДЭ по специальности 09.02.07.  
Пример задания: портал «Корочки.есть» — уже заложен в `exam_config.py`, но всё можно перенастроить под другое ТЗ.

> Сначала установка → потом запуск → потом настройка цветов и текстов под задание.

> **Другое задание (не «Корочки»)?** Читайте **[КАК_ПОДГОНЯТЬ_ПОД_ЗАДАНИЕ.md](КАК_ПОДГОНЯТЬ_ПОД_ЗАДАНИЕ.md)** — как менять БД, поля, другую фирму и сферу.

> **Архив для другого ПК:** файл `gia_exam_template_DEMO.zip` (рядом с папкой проекта) или соберите заново: `python create_archive.py`

## 2. Установка на Windows (пошагово)

### Шаг 1. Открыть папку проекта

```powershell
cd D:\Projects\gia_exam_template
```

(замените путь, если проект лежит в другом месте)

### Шаг 2. Создать виртуальное окружение

Виртуальное окружение — это «отдельная коробка» с библиотеками только для этого проекта.

```powershell
python -m venv .venv
```

### Шаг 3. Включить окружение

```powershell
.venv\Scripts\activate
```

В начале строки появится `(.venv)` — значит всё правильно.

> **Важно:** каждый раз, когда открываете новый терминал, снова выполняйте эту команду из папки проекта.

### Шаг 4. Установить библиотеки

```powershell
pip install -r requirements.txt
```

Подождите, пока всё скачается (нужен интернет).

### Шаг 5. Создать базу данных

```powershell
python manage.py makemigrations portal
python manage.py migrate
```

Появится файл `db.sqlite3` — это ваша база данных. Пароль к ней **не нужен**.

### Шаг 6. (По желанию) Django Admin

Только если нужен технический раздел `/django-admin/`:

```powershell
python manage.py createsuperuser
```

Введите логин, email (можно любой) и пароль — **запомните их**.

### Шаг 7. Запуск

```powershell
python manage.py runserver
```

Откройте в браузере: **http://127.0.0.1:8000/**

Остановить сервер: **Ctrl+C** в терминале.

## 3. Установка на Linux (пошагово)

Подходит для Ubuntu, Debian, Linux Mint, Astra Linux и похожих систем.

### Шаг 1. Установить Python и venv (один раз)

**Ubuntu / Debian:**
```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git
```

### Шаг 2. Перейти в папку проекта

```bash
cd путь проекта
```

(или путь, куда вы скопировали проект)

### Шаг 3. Создать и включить виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

После `source` в начале строки появится `(.venv)`.

> **Каждый новый терминал:** снова `cd` в папку проекта и `source .venv/bin/activate`.

### Шаг 4. Установить библиотеки

```bash
pip install -r requirements.txt
```

### Шаг 5. Создать базу данных

```bash
python manage.py makemigrations portal
python manage.py migrate
```

### Шаг 6. (По желанию) Django Admin

```bash
python manage.py createsuperuser
```

### Шаг 7. Запуск

```bash
python manage.py runserver 0.0.0.0:8000
```

- На этом же компьютере: **http://127.0.0.1:8000/**
- С другого устройства в сети: **http://IP_ЭТОГО_КОМПЬЮТЕРА:8000/**

Остановить: **Ctrl+C**.

### Linux: если `pip` ругается на права

Не используйте `sudo pip`. Работайте только внутри `.venv` (после `source .venv/bin/activate`).

### Linux: PostgreSQL на экзамене

```bash
pip install psycopg2-binary
```

Настройки подключения — в `config/settings.py` (см. раздел 10).

## 5. Две разные «админки» — не путать

Это самая частая путаница!

| | Панель из **задания** | **Django** Admin |
|---|----------------------|------------------|
| **Адрес** | `/admin-panel/login/` | `/django-admin/` |
| **Логин** | `Admin` | Тот, что вы задали в `createsuperuser` |
| **Пароль** | `KorokNET` | Тот, что вы задали в `createsuperuser` |
| **Где менять** | файл `exam_config.py` | команда `createsuperuser` |
| **Зачем на экзамене** | **Основная** — смена статусов заявок | Просмотр таблиц, ER-диаграмма |

> Логин и пароль с **регистрации** (`student1` и т.д.) **не подходят** для `/admin-panel/` — там только `Admin` / `KorokNET` из задания.

## 6. Где менять цвета и дизайн

### Пользовательская часть (синий портал)

Файл: **`static/css/main.css`**

В начале файла блок `:root` — главные цвета:

```css
:root {
    --portal-primary: #1a5fb4;   /* основной синий (шапка, кнопки) */
    --portal-accent: #3584e4;    /* второй оттенок (градиент шапки) */
    --portal-bg: #f6f8fc;        /* фон страницы */
}
```

| Что менять | Где в `main.css` |
|------------|------------------|
| Цвет шапки (navbar) | `--portal-primary`, `--portal-accent` |
| Фон сайта | `--portal-bg` |
| Бейджи статусов заявок | классы `.badge.status-new`, `.status-in_progress`, `.status-completed` |
| Анимация появления | `.fade-in` и `@keyframes fadeIn` |
| Слайдер на регистрации | `.hero-slider`, `.slider-btn` |

**Как подобрать цвет:** откройте [htmlcolorcodes.com](https://htmlcolorcodes.com/) — скопируйте код вида `#1a5fb4`.

**Логотип и картинки слайдера:** папка `static/images/`  
- `logo.svg` — иконка в шапке  
- `slide1.svg` … `slide4.svg` — слайдер (замените на PNG/JPG из архива задания)

**Шаблон шапки и меню:** `portal/templates/portal/base.html`

**Кнопки Bootstrap** (зелёная «Новая заявка» и т.д.) — классы в HTML: `btn-primary`, `btn-success`, `btn-outline-primary`.  
Справка по цветам Bootstrap: [getbootstrap.com/docs/5.3/customize/color](https://getbootstrap.com/docs/5.3/customize/color/)

## 7. Где менять тексты и логику под задание

### Главный файл настроек — `exam_config.py`

Откройте в редакторе и меняйте значения:

| Параметр | Что это |
|----------|---------|
| `SITE_NAME` | Название сайта в шапке |

| `ADMIN_LOGIN`, `ADMIN_PASSWORD` | Вход в `/admin-panel/` |

| Файл | Когда менять |
|------|--------------|

## 8. Частые ошибки и как мы их чинили

### Ошибка 1: «Неверный логин или пароль» после регистрации

**Симптом:** зарегистрировались, но войти не получается.

**Причина:** пароль сохранялся в БД без шифрования (баг в старой версии формы).

**Решение (уже исправлено в проекте):** в `portal/forms.py` поле `password` убрано из `Meta.fields`.

**Если старый пользователь всё ещё не входит:**
```bash
python manage.py fix_passwords --username student1 --password password1
```
или зарегистрируйтесь заново с **новым** логином.

---

### Ошибка 2: Не входит в `/admin-panel/` с логином `student1`

**Это не ошибка.** Панель из задания использует только **`Admin`** / **`KorokNET`** (см. раздел 5).

---

### Ошибка 3: `makemigrations` / «changes that are not yet reflected in a migration»

**Решение:**
```bash
python manage.py makemigrations portal
python manage.py migrate
```

Если база пустая и ничего не жалко:
```bash
# Windows
del db.sqlite3
# Linux
rm db.sqlite3

python manage.py migrate
```

---

### Ошибка 4: Django Admin падает на Python 3.14

**Симптом при открытии `/django-admin/`:**
```
AttributeError: 'super' object has no attribute 'dicts'
```

**Причина:** Python **3.14** + Django **5.1** несовместимы. Django 5.2+ исправляет это официально.

**Решение 1 (уже в проекте):** патч `config/py314_patch.py` подключается автоматически. Просто **перезапустите сервер**:
```bash
python manage.py runserver
```

**Решение 2 — обновить Django:**
```bash
pip install "Django>=5.2,<5.3"
```

**Решение 3 — сменить Python (рекомендуется для экзамена):**

*Windows:*
```powershell
py -3.12 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
```

*Linux:*
```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
```

| Python | Django 5.1 | Django 5.2+ | Патч в проекте |
|--------|------------|-------------|----------------|
| 3.12 / 3.13 | ✅ | ✅ | не нужен |
| 3.14 | ❌ без патча | ✅ | ✅ уже есть |

---

### Ошибка 5: Bootstrap не грузится (экзамен без интернета)

Скачайте Bootstrap 5 с [getbootstrap.com](https://getbootstrap.com/) в папку `static/bootstrap/`.

В `portal/templates/portal/base.html` и `base_admin.html` замените ссылки CDN на:
```html
{% load static %}
<link href="{% static 'bootstrap/css/bootstrap.min.css' %}" rel="stylesheet">
<script src="{% static 'bootstrap/js/bootstrap.bundle.min.js' %}"></script>
```

## 10. Остальное

### PostgreSQL вместо SQLite

В `config/settings.py` замените блок `DATABASES`:

```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.postgresql",
        "NAME": "имя_базы",
        "USER": "логин",
        "PASSWORD": "пароль",
        "HOST": "127.0.0.1",
        "PORT": "5432",
    }
}
```

```bash
pip install psycopg2-binary
python manage.py migrate
```
